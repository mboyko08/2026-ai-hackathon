# Task 11 — End-to-End Demo

## Objective

Run the complete system end-to-end, introduce one intentional mismatch, and confirm the full agent workflow detects and explains it.

## Deliverables

- Working demo that completes all ten steps below
- Validation report produced by the agent workflow
- Documented mismatch and confirmed root cause output

## Demo Steps

1. Run the PostgreSQL stored procedure to generate new OLTP records.
2. Run the SQL Server stored procedure to generate new OLTP records.
3. Confirm CDC change records appear in both source change tables.
4. Run the replication pipeline to extract and stage CDC changes.
5. Run the warehouse merge to apply staged changes to the target table.
6. **Introduce one intentional mismatch** — choose one:
   - Skip a CDC batch by manually advancing the pipeline checkpoint without loading
   - Insert a duplicate key into the warehouse target table
   - Null out a required column on one warehouse row
7. Ask the Orchestrator Agent: *"Validate the latest CDC replication from OLTP to the PostgreSQL warehouse."*
8. Validation Agent selects all six checks and invokes the CDC Executor skill.
9. CDC Executor skill returns structured results with at least one failed check.
10. Root Cause Agent reads centralized logs and returns a root cause summary with cited evidence and a recommended action.

## Acceptance Criteria

- [ ] All data flows from stored procedures through CDC capture, pipeline, and merge without errors
- [ ] Centralized log area contains database, pipeline, and warehouse log rows for the demo run
- [ ] At least one validation check returns status = 'failed'
- [ ] The failed check correctly identifies the intentional mismatch
- [ ] Root Cause Agent identifies the correct failure category
- [ ] Root Cause Agent cites actual log evidence (not invented)
- [ ] Root Cause Agent provides one specific recommended action
- [ ] Final Orchestrator Agent summary includes: overall status, failed checks, likely root cause, recommended action

## Test Cases

| # | Mismatch Scenario | Expected Detection |
| --- | --- | --- |
| T1 | Skipped CDC batch (checkpoint advanced without loading) | `checkpoint` check fails; Root Cause Agent identifies 'Stale pipeline checkpoint' |
| T2 | Duplicate key inserted into warehouse | `duplicate_keys` check fails; Root Cause Agent identifies 'Duplicate handling' |
| T3 | Required column nulled in warehouse | `null_counts` check fails; Root Cause Agent identifies 'Null handling' |
| T4 | No mismatch (clean run) | All six checks pass; Orchestrator Agent returns overall status = 'passed' |

## Demo Success Criteria

The demo is successful if:

- Source data was generated by stored procedures only (no manual inserts)
- CDC changes were captured or simulated from both OLTP sources
- CDC data was replicated into the warehouse via the pipeline
- All six validation checks ran
- At least one controlled mismatch was detected
- Centralized logs were used to explain the root cause
- A clear recommended fix was produced
